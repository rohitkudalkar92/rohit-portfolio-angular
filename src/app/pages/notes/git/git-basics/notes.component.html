• Working Directory: Your actual files and folders
  • Staging Area: Files ready to be committed
  • Repository: Committed snapshots of your project
  • Commit: A snapshot of your project at a specific time
  • Branch: Independent line of development
  • --global: Settings for all repositories on your system
  • --local: Settings for current repository only
  • --system: Settings for all users on the system
  • Commit often: Make small, focused commits
  • Write clear messages: Describe what and why
  • Check status: Always know what's staged
  • Review changes: Use git diff before committing
